## Project in brief

VVUQ-02 models a 60-second 8-phase pancreaticoduodenectomy (Whipple) performed by a
single autonomous humanoid surgeon, the clearly labeled hypothetical 2030 Unitree
H2-Surgical 1.0 (2 x 7-DOF arms, 2 x 20-DOF dexterous hands, 71 total DOF), with its
own two hands and no teleoperation. The deliverable is 10 humanoid-specific VVUQ
gates, each a fully specified Verify then Validate then Quantify gate bound to an
independent reference and to published external standards. The code generation step
(v0.7.0) produced the standalone codebase; the execution step (v0.8.0) ran every
entry point, the 172-test suite, the 10-gate decision surface, the 32-iteration
sweep, the 4-entrant tournament, and the full safety surface, and captured the
verbatim outputs. The image instruction step (v0.9.0) specified the publication
figures. This image generation step (v1.0.0) renders them.

## Thesis link

The cancer-automated thesis is that the robotic code assurance process, not code
generation, is the substantial and decision bearing part of the AI workflow, holding
VVUQ to a higher standard than code itself. These safety measures will ensure
upcoming physical AI oncology trial deliverables are faster, less expensive, and more
beneficial towards patients than conventional verifications.

The 15 figures make that thesis visible. The set is weighted toward the assurance
machinery, the 10 gates, their thresholds, the standards binding, the test budget,
and the decision surface, because that is where the thesis lives. The gate funnel,
the threshold forest, the standards binding matrix, the test treemap, the decision
parity, and the gated composite all place the assurance layer at the visible center
of the platform.

## How each figure was generated

Every figure was produced by Claude Code Opus 4.7 (1M context) Max from its v0.9.0
instruction with no manual positioning. For each numbered figure `NN-name`:

1. Read the instruction at `papers/VVUQ-02/image-instruct/NN-name/README.md` in full,
   including its data tables, its layout specification, and its acceptance checklist.
2. Read the shared conventions in `papers/VVUQ-02/image-instruct/README.md` (the page
   frame, palette, fonts, and the symbol and dash rules) so the figure matches every
   other figure in the set.
3. Confirm the grounding values against the cited source files (the v0.7.0 codegen
   modules, configs, and results, and the v0.8.0 execution sections and artifacts) by
   direct lookup, not estimation.
4. Author one self-contained matplotlib script at `imagegen/NN-name/NN-name.py`. The
   script hardcodes the grounded values as Python literals, reads no external data
   file, opens no network connection, and depends only on `matplotlib` and `numpy`.
   It sets a non-interactive backend with `matplotlib.use("Agg")`.
5. Build the figure to the shared page frame: portrait, full page, `figsize=(8.5,
   11)` inches, saved at `dpi=300`, which yields a 2550 by 3300 pixel image, on a
   deterministic `GridSpec` so nothing overlaps and no box needs nudging.
6. Render on a white background only. No dark mode. All text is dark on light and
   clearly legible.
7. Save to `imagegen/NN-name/NN-name.png` with `dpi=300` and `facecolor="white"`,
   without `bbox_inches="tight"` so the canvas stays the fixed full-page portrait
   size.
8. Verify the render against the instruction acceptance checklist: portrait and full
   size, every label present and aligned, the legend and the header and the footer
   inside their reserved bands, the section symbol rendered as `§` where required,
   and only single hyphens in any visible text.
9. Confirm the script passes `ruff check` and `ruff format --check` under the
   repository `ruff.toml`, so the `lint-and-format` CI job stays green across Python
   3.10, 3.11, and 3.12.
10. Commit the script and the PNG together for that figure, then continue to the next.

A shared `add_frame` helper auto-fits the header title, subtitle, and footer to the
page width so long captions never clip, and every script renders deterministically
(any random jitter is seeded with `np.random.default_rng(20260525)`).

## Directory and file naming conventions

| Purpose | Exact path |

| Matplotlib script (here) | `papers/VVUQ-02/imagegen/NN-name/NN-name.py` |
| 300 dpi image (here) | `papers/VVUQ-02/imagegen/NN-name/NN-name.png` |

`NN` is the zero padded two digit index from `01` to `15`. `name` is the short
hyphenated slug shown in the index below. The instruction subdirectory and the
imagegen subdirectory share the identical `NN-name` so a reader can move between the
specification and the rendered output without guessing.

## Standard output conventions

These conventions are shared by all 15 figures, so they form one coherent set.

### Page frame (portrait, full size)

- Orientation portrait, full page. `fig = plt.figure(figsize=(8.5, 11))`.
- Resolution 300 dpi on save, giving 2550 by 3300 pixels.
- Background white only, no dark mode.
- Reserved bands, as figure fractions: header `y` in `[0.93, 1.00]`, content `y` in
  `[0.07, 0.91]`, footer `y` in `[0.00, 0.06]`, left margin `x = 0.06`, right margin
  `x = 0.94`.

### Typography

- Font family `DejaVu Sans` (the matplotlib default, always present, no install).
- Title 20 pt bold (auto-fit smaller when a caption is long), subtitle 12.5 pt, panel
  or section headers 13.5 to 14 pt bold, body and data labels 11 pt, small
  annotations and the footer 9 pt.
- Text color near black `#1A1A1A` for body text on white, so contrast is high.

### Shared professional palette

| Role | Hex | Used for |
|------|-----|----------|
| Primary navy | `#1F3A5F` | primary structures, baselines, headers |
| Slate blue | `#4C72B0` | secondary structures, neutral series, left hand |
| Teal | `#2A9D8F` | process flow, automation, positive movement, right hand |
| Accept green | `#2E7D32` | accepted, passed, shippable, within bound |
| Block red | `#C0392B` | blocked, failed, over budget, breach |
| Escalate amber | `#E1A93B` | escalated to human, caution, gate, soft warning |
| Autonomy purple | `#6A4C93` | on-prem LLM intent, autonomy, lineage |
| Clinical rose | `#B5566E` | clinical baselines, patient framing |
| Neutral gray | `#6B7280` | connectors, idle state, secondary text |
| Panel fill | `#F4F6F8` | light card and panel backgrounds |
| Gridline gray | `#D9DEE3` | gridlines and thin separators |
| Near black | `#1A1A1A` | all primary text |

The three immediate-catastrophe gates (06 vascular no-fly, 09 shared-OR collision, 10
fault e-stop) are consistently flagged with the block red accent so they read the
same in every figure.

### Symbols and dashes

- The section symbol renders as `§` (U+00A7) wherever a figure references a paper
  section, an execution section, or a regulatory clause (for example `execution §03`,
  `ASME V&V 40 §8`). It is never spelled `SS` or `Section` where the symbol is meant.
- Only single hyphens appear in visible figure text. There are no em dashes, en
  dashes, double dashes, or triple dashes in any title, label, annotation, legend,
  header, or footer.
- Factors use a multiplication style `x` (for example `2.5x`); ranges use the word
  `to` (for example `38 to 43 mm`).

### Footer (every figure)

`cancer-automated v1.0.0  |  source: <grounding paths>  |  white background, 300 dpi,
portrait`, at `y = 0.03`, 9 pt, neutral gray. The grounding paths are listed per
figure.

## Reproduction and dependency note

Each script is pure `matplotlib` plus `numpy` (already a core dependency) and sets
`matplotlib.use("Agg")`, so it renders headless without a display:

```
pip install matplotlib
python papers/VVUQ-02/imagegen/NN-name/NN-name.py
```

The script writes its PNG to its own subdirectory. The core CI does not install
`matplotlib` and does not import `imagegen/`; the `lint-and-format` job runs `ruff`
and `yamllint` statically and the `test` job runs `pytest tests/`, so the scripts
keep CI green across Python 3.10, 3.11, and 3.12 as long as they are ruff clean, which
they are.

## Responsible use

These figures describe an automated assurance platform and a hypothetical 2030
humanoid deployment reference. They are planning and documentation artifacts.
Generated instructions, code, images, and papers are drafts: the 10 VVUQ gates plus a
recorded human reviewer must clear any deliverable before clinical use. The
H2-Surgical 1.0 is a clearly labeled hypothetical 2030 platform and every number is a
simulation result; the comparison is simulation against simulation. Deployment would
require IEC 80601-2-77, IEC 60601, ISO 13482, FDA SaMD Class III clearance, IRB
approval, and regulatory authorization. Only `kevinkawchak/cancer-automated` is edited
by this work.
